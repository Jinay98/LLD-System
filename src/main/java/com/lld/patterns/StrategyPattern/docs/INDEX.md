# Strategy Pattern - Algorithm Selection

Pluggable algorithms that can be swapped at runtime.

## 📚 Files
- **README.md** - Strategy Pattern theory and examples

## 🎯 When to Use Strategy
- Multiple algorithms for same task
- Algorithm selection at runtime
- Want to avoid if-else chains
- Easy testing of individual algorithms

## 🔄 Real-World Examples
- Sorting algorithms
- Compression methods
- Payment methods (Zomato example!)
- Notification channels

## 🔑 Key Takeaway
**Strategy Pattern:** Encapsulate algorithms, swap at runtime, no if-else
